PART A Input-Output Organization
1. I/O Device Interface
Definition: An I/O device interface is a hardware circuit that provides a pathway for data transfer between the CPU/memory and peripheral devices. It bridges the speed and signal differences between fast internal bus and slow external devices.

Why interfaces are needed: Peripherals operate at very different speeds and use different data formats than the CPU. A dedicated interface handles buffering, signal conversion, status checking, and control signaling.

Key Components of an I/O Interface:
  • Data Register – Temporarily holds data being transferred
  • Status Register – Indicates device state (busy, ready, error)
  • Control Register – Stores commands sent by CPU to device
  • Address Decoder – Selects the correct I/O port

2. I/O Transfers
Definition: I/O transfer is the process of moving data between an I/O device and the CPU or memory. Three main methods exist, differing in who controls the transfer and how CPU time is used.
2A. Programmed I/O (Polling / Busy Wait)
Definition: The CPU directly controls I/O transfer. It continuously checks (polls) the status register of the device until it is ready, then performs the transfer.

Working:

  1. CPU issues an I/O command to the interface
  2. CPU reads the status register in a loop (busy-wait)
  3. When device sets the "done/ready" flag, CPU reads/writes data register
  4. CPU continues with next instruction
AdvantageDisadvantage
Simple to implementCPU wastes time polling (busy-wait)
No special hardware neededVery inefficient for slow devices
Predictable timingCPU cannot do other work during I/O
2B. Interrupt-Driven I/O
Definition: The CPU initiates an I/O operation and then continues executing other instructions. When the device is ready, it sends an interrupt signal to the CPU. The CPU suspends its current task, executes an Interrupt Service Routine (ISR), and then resumes.

Working:

  1. CPU sends I/O command and resumes other work
  2. Device performs operation independently
  3. Device sends interrupt signal on the interrupt line
  4. CPU finishes current instruction, saves state (registers, PC) onto stack
  5. CPU jumps to ISR address (from Interrupt Vector Table)
  6. ISR handles data transfer
  7. CPU restores state (IRET) and resumes original program
Interrupt-driven I/O is far more efficient than programmed I/O because the CPU is free to do useful work between I/O operations.
AdvantageDisadvantage
CPU not idle during I/OInterrupt handling adds overhead
Good for low-speed devicesStill requires CPU for each data word
Supports multiple devicesContext-save/restore needed
2C. Direct Memory Access (DMA)
Definition: DMA allows an I/O device to transfer data directly to/from main memory without CPU involvement for each byte. A dedicated DMA Controller (DMAC) manages the transfer.

Working:

  1. CPU programs DMAC: starting address, count, direction (read/write)
  2. CPU sends I/O command to device and resumes other work
  3. DMAC requests bus control from CPU (Bus Request signal)
  4. CPU grants bus (Bus Grant signal) — CPU halts briefly (cycle stealing)
  5. DMAC transfers entire block directly between device ↔ memory
  6. DMAC sends interrupt to CPU when block transfer is complete
DMA Modes:
  • Burst Mode – Transfers entire block; CPU is halted throughout
  • Cycle Stealing – Steals one bus cycle at a time; CPU interleaved
  • Transparent Mode – Transfers only when CPU is not using the bus (no CPU slowdown, but complex)

Exam Tip: Know the working steps of all three modes and be able to draw a timing diagram. DMA is the most-asked in exams — remember the terms: Bus Request, Bus Grant, Cycle Stealing.
3. Privileged and Non-Privileged Instructions
Privileged Instructions: Instructions that can only be executed when the CPU is in Supervisor (Kernel) Mode. They control sensitive hardware resources. If a user-mode program tries to execute them, a privilege violation exception is raised.
Non-Privileged (User) Instructions: Instructions that can be executed in User Mode by any application program. They do not directly access hardware or critical system resources.

Mode Bit: Modern CPUs have a mode bit in the PSW (Program Status Word). 0 = Kernel Mode, 1 = User Mode. The OS switches to user mode when starting a user process and returns to kernel mode on interrupts/system calls.
This dual-mode operation is the foundation of OS protection. Without it, any program could crash the system or corrupt another program's data.
4. Software Interrupts (Traps / System Calls)
Definition: A software interrupt (also called a trap or exception) is an interrupt generated by the execution of a special instruction (e.g., INT n in x86) or by an exceptional condition in the program, rather than by external hardware.

Common Causes of Software Interrupts:

  • System Calls (SVC/INT) – User program requests OS services (e.g., file I/O, process creation)
  • Division by Zero – Arithmetic exception
  • Invalid Opcode – CPU encounters an unrecognized instruction
  • Page Fault – Accessed page not in memory (virtual memory)
  • Overflow – Arithmetic result too large for register
  • Privilege Violation – User mode attempts privileged instruction
